How it accesses your Claude usage

Where does it read data from?

Claude Code stores session logs locally at ~/.claude/projects/. Each session is a JSONL file with your conversation turns, tool usage, and token counts.

NarrowScore reads these files directly on your machine. It also checks ~/.claude/CLAUDE.md, memory files, and your settings for hooks/MCP config.

Does my data leave my machine?

No. The scan runs 100% locally. Your sessions, code, and conversations are never sent anywhere. The only thing that leaves your machine is your score — if you choose to publish it to the leaderboard.

How does it calculate spend?

It reads the actual message.usage fields from your session files — real input tokens, output tokens, cache tokens — and applies Claude's published pricing per model (Sonnet, Opus, Haiku). If your sessions don't have token data, it falls back to estimates based on turn count.

What gets published to the leaderboard?

Only if you run narrowscore publish: your username, score, tier, total spend, token count, and top tools. No session content, no code, no conversations.
